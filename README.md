# CameraToolKit - 工业相机图像标定与校正工具

基于图像标定与校正工具，提供传感器特性表征、快速色彩标定、色彩特性标定、光谱特性标定、色彩误差计算、坏点校正、平场校正功能。

## 功能概览

| 功能模块      | 说明                                                                  |
| --------- | ------------------------------------------------------------------- |
| 传感器特性表征   | 传感器特性分析模块，基于 EMVA 1288 的传感器特性表征 (EMVA 1288 Sensor Characterization) |
| 快速色彩标定    | 标定相机色彩参数，使相机在任意照明条件下实现标准值或参考图像的色彩效果                                 |
| 色彩特性标定    | 多光源环境下的白平衡和CCM标定，支持量化系数调节                                           |
| 光谱特性标定    | 生成光谱特性标定文件，用于相机自动色彩校正，支持出厂/用户标定模式                                   |
| 色彩误差计算    | 计算校正结果与参考图像的CIE DeltaE00色彩偏差                                        |
| 坏点校正      | 自动检测和校正传感器坏点（亮坏点/暗坏点），支持多帧统计确认                                      |
| 平场校正(LSC) | 镜头阴影校正，消除渐晕效应导致的中心亮边缘暗问题                                            |

## 新增特性

| 特性                 | 说明                                        |
| ------------------ | ----------------------------------------- |
| 12-bit/16-bit高位深支持 | 支持处理工业相机的高位深原始图像                          |
| Tone Mapping       | 多种映射模式（线性/百分位/直方图均衡化/伽马）将16-bit映射到8-bit显示 |
| wxAuiManager布局     | 可停靠的三栏布局，支持浮动、拖拽调整                        |
| 日志系统               | 自动记录操作、数据和错误信息，便于调试和排查问题                  |
| 跨平台支持              | 支持Windows和Mac系统                           |
| 校正Pipeline         | 链式图像校正处理（LSC→降噪→插值→白平衡→CCM→Gamma）         |
| 内存池机制              | 预分配图像缓冲区，避免高帧率采集时的内存碎片                    |
| 多线程处理              | 使用wxThread异步执行耗时算法，保持UI响应                 |

### C++ ABI 兼容性

**重要**：由于 MSVC 运行时库选择会影响 C++ ABI 兼容性，项目已在 CMakeLists.txt 中显式配置：

```cmake
if(MSVC)
    set(CMAKE_MSVC_RUNTIME_LIBRARY "MultiThreaded$<$<CONFIG:Debug>:Debug>DLL")
endif()
```

**运行时库选项说明**：

| 选项                            | 含义         | 动态链接 | 适用场景                          |
| ----------------------------- | ---------- | ---- | ----------------------------- |
| `MultiThreadedDLL` (MD)       | 多线程动态运行时   | 是    | 与 wxWidgets/OpenCV 预编译库兼容（推荐） |
| `MultiThreadedDebugDLL` (MDd) | 多线程动态调试运行时 | 是    | Debug 模式                      |
| `MultiThreaded` (MT)          | 多线程静态运行时   | 否    | 独立部署，不依赖 VC++ 运行时 DLL         |
| `MultiThreadedDebug` (MTd)    | 多线程静态调试运行时 | 否    | Debug 模式，独立部署                 |

**注意事项**：

- wxWidgets 通过 vcpkg 安装时默认使用 **MD**（动态链接）
- OpenCV 预编译库通常也使用 **MD**
- 确保项目与所有依赖库使用**相同的运行时库**，否则会出现链接错误或运行时异常
- 如需使用静态链接（MT），需自行编译 wxWidgets 和 OpenCV

## 功能详细说明

### 1. SCM（传感器特性分析模块）

**功能说明**：SCM（Sensor Characterization Module）基于 EMVA 1288 标准，对图像传感器进行全面的性能评估和特性分析。

**评估指标**：

| 指标   | 符号     | 说明              | 单位       |
| ---- | ------ | --------------- | -------- |
| 系统增益 | K      | 每个数字值对应的光子数     | e-/DN    |
| 暗噪声  | σ_dark | 传感器在无光照条件下的噪声水平 | e-       |
| 满阱容量 | FWC    | 像素能容纳的最大电子数     | e-       |
| 动态范围 | DR     | 传感器可测量的亮度范围     | dB / bit |
| 暗电流  | I_dark | 单位时间内像素产生的暗信号   | e-/s     |
| 信噪比  | SNR    | 信号与噪声的比值        | dB       |

**评估流程**：

1. **采集暗场图像**：在不同曝光时间下采集至少50帧暗场图像（关闭光源）
2. **采集亮场图像**：在不同曝光时间下采集至少50帧亮场图像（使用均匀面光源如积分球）
3. **计算统计量**：计算各曝光时间下的均值和方差
4. **拟合PTC曲线**：根据亮场数据拟合光子传递曲线（Photon Transfer Curve）
5. **推导系统增益**：从PTC曲线斜率计算系统增益
6. **计算性能指标**：基于增益和噪声数据计算所有性能指标

**操作方式**：

- **输入**：暗场图像文件夹路径 + 亮场图像文件夹路径 + 曝光时间（μs）
- **批量加载**：输入文件夹路径后，自动扫描并加载所有图像文件（支持 .png, .jpg, .jpeg, .tif, .tiff, .bmp）
- **多曝光支持**：可在不同曝光时间下加载多组图像，评估暗电流随曝光时间的变化

**输出结果**：

- **报告导出**：生成完整的评估报告（txt格式）
- **JSON导出**：导出结构化的评估结果（JSON格式）
- **数据指标**：
  - 系统增益（e-/DN 和 DN/e-）
  - 暗噪声（e- 和 DN）
  - 暗信号（DN）
  - 暗电流（e-/s 和 e-/s/mm²）
  - 满阱容量（e- 和 DN）
  - 动态范围（dB 和 bit）
  - 信噪比曲线（10%/50%/90%信号时的SNR及峰值SNR）

**参考标准**：EMVA Standard 1288 - Standard for Characterization of Image Sensors

### 2. 快速色彩标定

**功能说明**：标定相机色彩参数，使相机在任意照明条件下实现标准值或参考图像的色彩效果。

**标定模式**：

- **标准模式**：使用X-Rite Classic 24色卡sRGB D65标准参考值，使相机实现标准值的色彩效果
- **参考模式**：导入自定义参考图像，使相机实现与参考图像相同的色彩效果

**操作流程**：

1. 采集/导入图像
2. 绘制24色卡ROI
3. 检查亮度值（建议范围0.55~0.65，对应24色卡最后一行第二个色块的G通道数值）
4. 若亮度不在范围内，可点击"自动"调节曝光和增益
5. 选择标定模式并导入参考（参考模式需导入参考图像）
6. 高级设置：RGB统计方法（中值法/均值法）、白平衡标定方法（平均法/解析法）、CCM标定方法（最优/最小二乘）
7. 执行标定，查看白平衡参数、CCM矩阵和DeltaE00误差

**核心算法**：

- 白平衡标定：平均法（各色块分别计算后取平均）/ 解析法（误差最小原则）
- CCM标定：3x3色彩校正矩阵，支持最优法（SVD分解）和最小二乘法
- 亮度检查：G通道归一化值，灰度换算 [255×0.55] ~ [255×0.65]

### 3. 色彩特性标定

**功能说明**：在多种光源环境下进行白平衡和CCM标定，生成色彩特性标定文件。

**标定流程**：

1. 采集各光源下24色标准色卡的色彩特性
2. 每个光源需填写名称和色温，绘制24色ROI获取当前亮度
3. 添加光源信息，更换光源重复采集
4. 设置标定参数（白平衡方法、CCM方法、标准色彩选择）
5. 预览标定结果，启用量化开关可调整白平衡和CCM的量化系数
6. 导出标定文件

**量化系数**：V5.0.0新增功能，支持对色彩校正参数进行对应倍数的量化，便于硬件运算。

### 4. 光谱特性标定

**功能说明**：生成光谱特性标定文件，用于实现相机的自动色彩校正功能。

**标定模式**：
| 模式 | 适用场景 | 复杂度 |
|------|---------|--------|
| 出厂标定 | 从零开始标定典型光源 | 标准/简易 |
| 用户标定 | 基于出厂标定增加更多光源 | 标准/简易 |

**操作复杂度**：

- **标准**：需采集各种光源在标准24色背景下的反射光谱曲线（需准备24张不同颜色卡纸）
- **简易**：仅需采集白色背景下的反射光谱曲线（仅需白色卡纸）

**限制**：光谱标定文件最多记录20种光源数据。出厂标定通常使用8种光源，用户标定可额外添加12种。

**标定区域计算工具**：

- 方法1：已知镜头焦距和工作距离
- 方法2：已知物方视野范围

### 5. 色彩误差计算

**功能说明**：计算校正结果和参考图像的色彩偏差，评估校正结果是否可用。

**支持的误差指标**：

- **DeltaE00**（CIEDE2000）：最精确的色彩差异度量，考虑了人眼视觉特性
- **DeltaE76**（CIE76）：Lab空间欧氏距离
- **DeltaE94**（CIE94）：改进的色彩差异公式

**操作流程**：

1. 导入输入图像和参考图像
2. 绘制ROI（支持24色卡ROI、140色卡ROI、自定义ROI）
3. 可选：调整ROI提取的颜色、添加自定义颜色对、传入颜色文件（*.ini格式）
4. 可选：开启sRGB色彩空间（仅当输入和参考图像均为sRGB时）
5. 点击计算，查看各颜色对的DeltaE00误差
6. 导出误差结果

**误差评估参考**：

- DeltaE00 < 1.0：人眼不可察觉
- DeltaE00 1.0~3.0：可接受范围
- DeltaE00 3.0~6.0：明显差异
- DeltaE00 > 6.0：显著差异

### 6. 坏点校正

**功能说明**：自动检测和校正相机传感器上的坏点。

**坏点类型**：

- **亮坏点**：像素亮度异常高于邻域均值
- **暗坏点**：像素亮度异常低于邻域均值

**检测参数**：

- 阈值：与邻域均值的差值标准
- 平均亮度范围：限制计入计算的图像平均灰度范围，防止过曝
- 核大小：3×3或5×5邻域

**多帧统计**：对多张图像进行坏点计算时，仅当某坏点存在于每张图像中才被确认为真实坏点。

**校正方法**：

- 邻域均值插值
- 邻域中值插值

**数据管理**：

- 支持CSV格式导入/导出坏点信息
- 支持手动编辑坏点坐标
- 支持清空坏点列表

**支持格式**：Mono 8/10/12、BayerBG8/RG8/GR8/GB8、Bayer10/12

### 7. 平场校正（LSC）

**功能说明**：镜头阴影校正（Lens Shading Correction），消除镜头渐晕效应导致的中心亮、边缘暗问题。

**标定基准方式**：
| 方式 | 说明 |
|------|------|
| 中心为基准 | 以图像中心区域亮度为基准 |
| 最亮区域为基准 | 以图像最亮区域均值为基准 |
| 目标亮度 | 以指定亮度值为基准 |

**标定参数**：

- 边缘填充系数：避免边缘出现异常色块或明暗失真
- 平滑核大小：控制增益图的平滑程度
- 通道模式：独立通道校正 / 统一增益

**暗场校正**：支持使用暗场图像减去暗电流后计算增益，提高标定精度。

**均匀性指标**：校正后计算图像均匀性（1 - 变异系数），值越接近1表示越均匀。

### Tone Mapping模式

| 模式                     | 说明          | 适用场景            |
| ---------------------- | ----------- | --------------- |
| LINEAR                 | 线性映射全范围     | 已知数据范围时         |
| PERCENTILE             | 按百分位裁剪后线性映射 | **默认**，自动适应动态范围 |
| HISTOGRAM_EQUALIZATION | 直方图均衡化      | 对比度增强           |
| GAMMA                  | 伽马校正后映射     | 人眼感知优化          |

### 校正Pipeline

采用链式处理架构，各算法模块抽象为统一的 `ImageNode` 接口：

#### 降噪位置配置

**问题分析**：LSC（镜头阴影校正）会对图像进行增益放大，特别是边缘区域。如果噪声存在于 LSC 之前，LSC 的增益操作会放大边缘区域的噪声。

**降噪位置选项**：

| 位置           | 流程                              | 适用场景   | 优点                    | 缺点          |
| ------------ | ------------------------------- | ------ | --------------------- | ----------- |
| **Pre-LSC**  | `Denoise → LSC → ...`           | 高噪声传感器 | 先消除散粒噪声，避免 LSC 放大噪声   | 可能损失微弱信号细节  |
| **Post-LSC** | `LSC → Denoise → ...`           | 低噪声传感器 | 保留更多细节，消除 LSC 边缘放大的噪声 | LSC 会放大原始噪声 |
| **Both**     | `Denoise → LSC → Denoise → ...` | 复杂场景   | 兼顾噪声抑制和细节保留           | 计算量翻倍       |

**推荐策略**：

- 高噪声环境（如低光照、高ISO）：使用 **Pre-LSC** 或 **Both**
- 低噪声环境（如良好光照、低ISO）：使用 **Post-LSC**，保留更多高频细节

**默认流程**：`LSC → Denoise → Interpolation → WhiteBalance → CCM → Gamma`（Post-LSC）

支持**原地修改（In-place）**和**双缓冲机制**，降低内存开销。

## 算法原理

### 1. 白平衡算法（White Balance）

白平衡算法的核心目标是调整RGB三通道增益，使白色物体在不同光照条件下呈现为中性灰色。本系统实现了多种白平衡算法，支持基于色卡的标定和自动白平衡。

#### 1.1 平均法（Average Method）

对色卡中所有色块的RGB值分别取平均，以G通道为基准计算R、B通道的增益：

$$\overline{R} = \frac{1}{n} \sum_{i=1}^{n} R_i$$
$$\overline{G} = \frac{1}{n} \sum_{i=1}^{n} G_i$$
$$\overline{B} = \frac{1}{n} \sum_{i=1}^{n} B_i$$

$$R_{gain} = \frac{\overline{G}}{\overline{R}}, \quad G_{gain} = 1.0, \quad B_{gain} = \frac{\overline{G}}{\overline{B}}$$

**实现细节**：遍历所有色块的RGB值，分别累加后取平均值，再以G通道增益为1.0计算R、B通道的归一化增益。

**优点**：计算简单、稳健，对单个色块的异常值不敏感

**适用场景**：标准色卡标定模式，色卡颜色分布均匀

#### 1.2 解析法（Analytic Method）

基于误差最小化原则求解最优增益，最小化目标函数：

$$\min_{WB} \sum_{i=1}^{n} \| WB \cdot src_i - ref_i \|^2$$

当有参考颜色时，以参考颜色为目标；否则以G通道为基准：

$$R_{gain} = \frac{\sum ref_{R_i}}{\sum src_{R_i}}, \quad G_{gain} = 1.0, \quad B_{gain} = \frac{\sum ref_{B_i}}{\sum src_{B_i}}$$

**实现细节**：分别累加源颜色和参考颜色的RGB分量，按通道比值计算增益。有参考时以参考通道和为目标，无参考时以源G通道和为基准。

**优点**：利用参考信息，标定精度更高

**适用场景**：参考图像模式，需要匹配特定参考图像的色彩

#### 1.3 灰度世界算法（Gray World）

假设场景中所有颜色的平均为灰色，直接计算整幅图像的RGB均值：

$$\mu = \text{cv::mean}(image)$$
$$R_{gain} = \frac{\mu_G}{\mu_R}, \quad G_{gain} = 1.0, \quad B_{gain} = \frac{\mu_G}{\mu_B}$$

**实现细节**：使用OpenCV的`cv::mean()`函数直接计算整幅图像的RGB均值，然后以G通道为基准计算增益。

**适用场景**：场景颜色分布均匀时效果较好，如室内环境

#### 1.4 完美反射算法（Perfect Reflector）

假设图像中最亮的部分为理想白色，通过直方图阈值提取高亮区域：

```
1. 将图像转换为灰度图
2. 计算灰度直方图
3. 找到亮度前ratio%的像素阈值T（默认ratio=0.1）
4. 创建二值掩膜：gray >= T
5. 计算掩膜区域内的平均RGB
6. R_gain = mean[G] / mean[R], B_gain = mean[G] / mean[B]
```

**实现细节**：从直方图的高端开始累加，直到达到总像素数的ratio%，以此确定高亮区域阈值。使用`cv::mean(image, mask)`计算掩膜区域的加权均值。

**适用场景**：场景中存在白色参考物体时效果较好

#### 1.5 RGB统计方法

在提取ROI颜色时，支持两种统计方法：

| 方法          | 实现方式               | 特点                   |
| ----------- | ------------------ | -------------------- |
| 均值法（MEAN）   | `cv::mean(region)` | 计算简单，对异常值敏感          |
| 中值法（MEDIAN） | 排序后取中间值            | 抗干扰能力更强，避免极端像素影响（默认） |

**实现细节**：中值法需要遍历ROI内所有像素，按通道分别收集像素值，排序后取中间位置的值。

#### 1.6 应用校正

```cpp
cv::Mat WhiteBalance::apply(const cv::Mat& image, const WBCorrectParams& params) {
    // 转换为浮点型
    image.convertTo(result, CV_64F);

    // 逐像素应用增益（注意OpenCV的BGR顺序）
    pixel[0] *= params.b_gain;  // B通道
    pixel[1] *= params.g_gain;  // G通道
    pixel[2] *= params.r_gain;  // R通道

    // 截断到有效范围
    cv::min(result, max_val, result);
    cv::max(result, 0.0, result);

    return result;
}
```

**注意**：OpenCV默认使用BGR通道顺序，因此在应用增益时需要注意通道对应关系。

### 2. 色彩校正矩阵（CCM）算法

3×3色彩校正矩阵用于将相机RGB映射到标准RGB空间，补偿相机光谱响应与理想响应的差异。

#### 2.1 数学模型

$$\begin{bmatrix} R_{out} \\ G_{out} \\ B_{out} \end{bmatrix} = \begin{bmatrix} m_{00} & m_{01} & m_{02} \\ m_{10} & m_{11} & m_{12} \\ m_{20} & m_{21} & m_{22} \end{bmatrix} \begin{bmatrix} R_{in} \\ G_{in} \\ B_{in} \end{bmatrix} + \begin{bmatrix} offset_R \\ offset_G \\ offset_B \end{bmatrix}$$

其中 `offset` 通常为零向量，简化为：$$RGB_{out} = CCM \cdot RGB_{in}$$

**物理意义**：相机拍摄的RGB值是场景光谱与相机光谱响应函数的卷积，通过CCM矩阵可以将相机RGB线性映射到标准色彩空间（如sRGB）。

#### 2.2 Gamma预处理

在标定前对颜色进行Gamma校正，将非线性的RGB值转换为线性值，提升标定精度：

| 类型        | 公式                         | 说明            |
| --------- | -------------------------- | ------------- |
| NONE      | 无处理                        | 默认模式          |
| VALUE     | `RGB' = 255 × (RGB/255)^γ` | 应用指定Gamma值    |
| LRGB2SRGB | sRGB解码曲线                   | 将sRGB转换为线性RGB |

**sRGB解码公式**：

```
c ≤ 0.04045 → c / 12.92
c > 0.04045 → ((c + 0.055) / 1.055)^2.4
```

#### 2.3 标定方法

给定N个颜色对 `(src_i, ref_i)`，构建线性方程组 `A × ccm_flat = b`：

- **系数矩阵A**：大小为 `3N × 9`，每行对应一个颜色分量的方程
- **目标向量b**：大小为 `3N × 1`，存储参考颜色值

```
对于第i个颜色对（已归一化到0-1）：
A[3i, 0:2]     = [src_R_i, src_G_i, src_B_i]     → R输出方程
A[3i+1, 3:5]   = [src_R_i, src_G_i, src_B_i]     → G输出方程
A[3i+2, 6:8]   = [src_R_i, src_G_i, src_B_i]     → B输出方程

b[3i]     = ref_R_i
b[3i+1]   = ref_G_i
b[3i+2]   = ref_B_i
```

**实现细节**：每个颜色对贡献3个方程（分别对应R、G、B输出通道），矩阵A的每一行只有3个非零元素，对应输入RGB值。

#### 2.4 求解方法

| 方法                   | OpenCV求解器           | 特点                                  |
| -------------------- | ------------------- | ----------------------------------- |
| 最优法（OPTIMAL）         | `cv::DECOMP_SVD`    | 使用SVD分解求解最小二乘，数值稳定性好，精度高（默认）        |
| 最小二乘法（LEAST_SQUARES） | `cv::DECOMP_NORMAL` | 使用正规方程 `A^T A × x = A^T b` 求解，计算速度快 |

**SVD分解优势**：当系数矩阵A接近奇异或条件数较大时，SVD分解仍能稳定求解，避免正规方程的数值不稳定问题。

#### 2.5 应用校正

对每个像素执行矩阵乘法：

```cpp
// 归一化到0-1
cv::Vec3d p = pixel / 255.0;

// 应用CCM: RGB_out = CCM × RGB_in + offset
out[0] = ccm(0,0)*p[0] + ccm(0,1)*p[1] + ccm(0,2)*p[2] + offset[0];
out[1] = ccm(1,0)*p[0] + ccm(1,1)*p[1] + ccm(1,2)*p[2] + offset[1];
out[2] = ccm(2,0)*p[0] + ccm(2,1)*p[1] + ccm(2,2)*p[2] + offset[2];

// 反归一化并截断
out = clamp(out × 255.0, 0, 255);
```

#### 2.6 量化系数

支持对标定参数进行量化，便于硬件实现：

```
quantized_r_gain = round(r_gain × quant_coeff)
quantized_ccm[i][j] = round(ccm[i][j] × quant_coeff)
```

**常用量化系数**：1024（10位精度）、4096（12位精度）

### 3. CIEDE2000色彩误差算法

CIEDE2000是目前最精确的色彩差异计算公式，考虑了人眼对不同颜色区域的感知差异。算法分为三个步骤：

#### 3.1 Step 1：计算C'ab和h'ab

$$C_{1}^{'ab} = \sqrt{a_1^2 + b_1^2}, \quad C_{2}^{'ab} = \sqrt{a_2^2 + b_2^2}$$
$$C_{avg}^{'ab} = \frac{C_{1}^{'ab} + C_{2}^{'ab}}{2}$$

$$G = 0.5 \cdot \left(1 - \sqrt{\frac{(C_{avg}^{'ab})^7}{(C_{avg}^{'ab})^7 + 25^7}}\right)$$

$$a_1' = a_1 \cdot (1 + G), \quad a_2' = a_2 \cdot (1 + G)$$

$$C_1' = \sqrt{a_1'^2 + b_1^2}, \quad C_2' = \sqrt{a_2'^2 + b_2^2}$$

$$h_1' = \text{atan2}(b_1, a_1'), \quad h_2' = \text{atan2}(b_2, a_2') \quad (\text{转换到} \, 0^\circ \sim 360^\circ)$$

**G因子**：补偿低饱和度区域的感知差异，当饱和度接近0时，G接近0.5，对a通道进行放大。

#### 3.2 Step 2：计算ΔL'、ΔC'、ΔH'

$$\Delta L' = L_2 - L_1$$
$$\Delta C' = C_2' - C_1'$$

$$\Delta h' = \begin{cases} 
h_2' - h_1' + 360^\circ & \text{若} \, h_2' < h_1' \, \text{且} \, |h_2' - h_1'| > 180^\circ \\
h_2' - h_1' - 360^\circ & \text{若} \, h_2' > h_1' \, \text{且} \, |h_2' - h_1'| > 180^\circ \\
h_2' - h_1' & \text{否则}
\end{cases}$$

$$\Delta H' = 2 \cdot \sqrt{C_1' \cdot C_2'} \cdot \sin\left(\frac{\Delta h'}{2}\right)$$

**色调差异处理**：色调角在0°和360°处是连续的，需要特殊处理跨越边界的情况。

#### 3.3 Step 3：计算CIEDE2000

$$L_{avg}' = \frac{L_1 + L_2}{2}, \quad C_{avg}' = \frac{C_1' + C_2'}{2}$$

$$h_{avg}' = \begin{cases} 
\frac{h_1' + h_2'}{2} & \text{若} \, |h_1' - h_2'| \leq 180^\circ \\
\frac{h_1' + h_2' + 360^\circ}{2} & \text{若} \, h_1' + h_2' < 360^\circ \\
\frac{h_1' + h_2' - 360^\circ}{2} & \text{若} \, h_1' + h_2' \geq 360^\circ
\end{cases}$$

$$T = 1 - 0.17\cos(h_{avg}' - 30^\circ) + 0.24\cos(2h_{avg}') + 0.32\cos(3h_{avg}' + 6^\circ) - 0.20\cos(4h_{avg}' - 63^\circ)$$

$$S_L = 1 + \frac{0.015(L_{avg}' - 50)^2}{\sqrt{20 + (L_{avg}' - 50)^2}}$$
$$S_C = 1 + 0.045C_{avg}'$$
$$S_H = 1 + 0.015C_{avg}' \cdot T$$

$$R_T = -\sin\left(2 \cdot 30^\circ \cdot \exp\left(-\left(\frac{h_{avg}' - 275^\circ}{25^\circ}\right)^2\right)\right) \cdot 2 \cdot \sqrt{\frac{(C_{avg}')^7}{(C_{avg}')^7 + 25^7}}$$

$$\Delta E_{00} = \sqrt{\left(\frac{\Delta L'}{S_L}\right)^2 + \left(\frac{\Delta C'}{S_C}\right)^2 + \left(\frac{\Delta H'}{S_H}\right)^2 + R_T \cdot \left(\frac{\Delta C'}{S_C}\right) \cdot \left(\frac{\Delta H'}{S_H}\right)}$$

**各参数含义**：

- **S_L**：亮度权重因子，低亮度区域权重更高，反映人眼对暗部差异更敏感
- **S_C**：饱和度权重因子，高饱和度区域权重更高
- **S_H**：色调权重因子，与色调角相关，不同色调区域的感知差异不同
- **R_T**：旋转项，补偿蓝色区域（约275°）的感知异常，当色调角接近275°时产生负向修正

#### 3.4 RGB转Lab色彩空间

```
1. sRGB解码（如果启用）：
   c ≤ 0.04045 → c / 12.92
   c > 0.04045 → ((c + 0.055) / 1.055)^2.4

2. RGB → XYZ（sRGB D65矩阵）：
   $$\begin{bmatrix} X \\ Y \\ Z \end{bmatrix} = \begin{bmatrix} 0.4124564 & 0.3575761 & 0.1804375 \\ 0.2126729 & 0.7151522 & 0.0721750 \\ 0.0193339 & 0.1191920 & 0.9503041 \end{bmatrix} \begin{bmatrix} R \\ G \\ B \end{bmatrix}$$

3. XYZ归一化（D65白点）：
   X /= 0.95047, Y /= 1.00000, Z /= 1.08883

4. XYZ → Lab：
   f(t) = t^(1/3)                    (t > (6/29)^3)
   f(t) = t / (3 × (6/29)^2) + 4/29  (t ≤ (6/29)^3)

   L = 116 × f(Y) - 16
   a = 500 × (f(X) - f(Y))
   b = 200 × (f(Y) - f(Z))
```

**D65白点**：CIE标准照明体D65的色度坐标，代表平均日光，色温约6500K。

#### 3.5 其他误差指标

| 指标       | 公式                                                                                                                                         | 特点                       |
| -------- | ------------------------------------------------------------------------------------------------------------------------------------------ | ------------------------ |
| DeltaE76 | $$\Delta E_{76} = \sqrt{(\Delta L)^2 + (\Delta a)^2 + (\Delta b)^2}$$                                                                      | Lab空间欧氏距离，计算简单但感知不均匀     |
| DeltaE94 | $$\Delta E_{94} = \sqrt{\left(\frac{\Delta L}{S_L}\right)^2 + \left(\frac{\Delta C}{S_C}\right)^2 + \left(\frac{\Delta H}{S_H}\right)^2}$$ | 引入权重因子，比E76更精确，适用于工业色差检测 |

#### 3.6 误差评估参考

| DeltaE00范围 | 视觉感知     | 应用场景        |
| ---------- | -------- | ----------- |
| < 1.0      | 人眼不可察觉   | 专业色彩校正、高端印刷 |
| 1.0 ~ 3.0  | 轻微差异，可接受 | 一般摄影、显示器校准  |
| 3.0 ~ 6.0  | 明显差异     | 工业检测、质量控制   |
| > 6.0      | 显著差异     | 需要重新标定      |

### 4. 坏点检测与校正算法

坏点检测用于识别传感器上异常的像素，分为亮坏点（亮度异常偏高）和暗坏点（亮度异常偏低）。

#### 4.1 单帧检测

对于每个像素(x,y)，计算其与邻域均值的差值：

```
1. 计算邻域均值（排除中心像素）：
   sum = Σ(pixel(x+dx, y+dy))  for dx, dy ∈ [-k/2, k/2], (dx,dy) ≠ (0,0)
   mean = sum / (k² - 1)

2. 计算差值：diff = pixel(x,y) - mean

3. 判断坏点：
   亮坏点：diff > threshold
   暗坏点：diff < -threshold
```

**参数说明**：

| 参数                 | 默认值   | 说明                      |
| ------------------ | ----- | ----------------------- |
| kernel_size        | 3     | 邻域大小（3×3或5×5）           |
| threshold          | 30    | 检测阈值，与邻域均值的差值标准         |
| avg_brightness_min | 0.0   | 图像平均亮度最小值，低于此值不检测       |
| avg_brightness_max | 255.0 | 图像平均亮度最大值，高于此值不检测（防止过曝） |

**实现细节**：为避免过曝图像影响检测结果，先计算整幅图像的平均亮度，若不在指定范围内则跳过检测。

#### 4.2 多帧统计检测

为了排除噪声引起的误检，对多帧图像进行统计确认：

```
1. 对每帧图像分别检测坏点，记录候选坏点集合
2. 使用集合交集操作，找出所有帧中共同存在的坏点
3. 仅当坏点在每帧中都存在时才确认为真实坏点
```

**原理**：真实坏点是传感器硬件缺陷，在所有帧中都会出现；而噪声引起的伪坏点具有随机性，只会在部分帧中出现。

**实现细节**：使用`std::set`存储每帧的坏点坐标，通过集合交集操作找到共同的坏点。

#### 4.3 校正方法

| 方法     | 公式                              | 特点              |
| ------ | ------------------------------- | --------------- |
| 邻域均值插值 | `val = Σ(neighbors) / (k² - 1)` | 计算简单，平滑效果好      |
| 邻域中值插值 | `val = median(neighbors)`       | 抗干扰能力强，适合处理椒盐噪声 |

**实现细节**：

- **均值插值**：取周围8个（3×3邻域）或24个（5×5邻域）像素的平均值
- **中值插值**：收集邻域像素值，排序后取中间值，能有效避免异常像素的影响

#### 4.4 支持的像素格式

- Mono 8/10/12
- BayerBG8/RG8/GR8/GB8
- Bayer10/12

**处理方式**：对于Bayer格式图像，直接在原始Bayer数据上进行坏点检测和校正，无需先进行Demosaicing。

### 5. 平场校正（LSC）算法

平场校正（Lens Shading Correction）用于消除镜头渐晕效应，使图像亮度分布均匀。

#### 5.1 增益图计算

增益图表示每个像素需要乘以的校正系数：    

```
Gain(x,y) = Reference_Brightness / Flat(x,y)
```

**标定基准方式**：

| 方式      | 计算方法                 | 适用场景               |
| ------- | -------------------- | ------------------ |
| 中心为基准   | 取中心区域（半径为图像尺寸的1/8）均值 | 大部分场景，中心区域通常最亮（默认） |
| 最亮区域为基准 | 遍历图像取最大块均值           | 图像中心不一定最亮的特殊情况     |
| 目标亮度为基准 | 使用指定的目标亮度值           | 需要标准化到特定亮度值        |

**中心区域计算**：

```
center_radius = min(width, height) / 16
center_region = image[height/2 - r : height/2 + r, width/2 - r : width/2 + r]
reference_brightness = mean(center_region)
```

#### 5.2 暗场校正

当提供暗场图像时，先减去暗电流再计算增益：

```
Corrected_Flat(x,y) = Flat(x,y) - Dark(x,y)
Gain(x,y) = Reference_Brightness / Corrected_Flat(x,y)
```

**作用**：消除传感器暗电流噪声和固定模式噪声，提高标定精度

**注意**：暗场图像需要在相同曝光条件下拍摄（盖上镜头盖）

#### 5.3 增益图平滑

对增益图进行高斯平滑，避免校正后出现块状伪影：

```
Smoothed_Gain = GaussianBlur(Gain, (smooth_kernel, smooth_kernel), 0)
```

**参数说明**：

| 参数            | 默认值 | 说明               |
| ------------- | --- | ---------------- |
| smooth_kernel | 64  | 高斯核大小，值越大平滑效果越明显 |

**实现细节**：高斯核大小必须为奇数，实际使用时会自动调整为最近的奇数。

#### 5.4 边缘填充

使用镜像反射填充边缘区域，防止边缘出现异常：

```
Padded_Gain = copyMakeBorder(Smoothed_Gain, edge_fill, BORDER_REFLECT_101)
Gain = Padded_Gain[edge_fill:-edge_fill, edge_fill:-edge_fill]
```

**参数说明**：

| 参数        | 默认值 | 说明      |
| --------- | --- | ------- |
| edge_fill | 2   | 边缘填充像素数 |

**作用**：避免边缘像素因邻域不完整导致的增益异常，使用镜像反射可以保持边缘的连续性。

#### 5.5 通道模式

| 模式     | 说明           | 适用场景             |
| ------ | ------------ | ---------------- |
| 独立通道校正 | 每个通道使用独立的增益图 | RGB图像，各通道渐晕特性不同  |
| 统一增益   | 所有通道使用相同的增益图 | 灰度图像或需要保持色彩平衡的场景 |

#### 5.6 应用校正

$$Corrected(x,y) = Image(x,y) \cdot Gain(x,y)$$

使用逐像素乘法（`cv::Mat::mul`）实现高效校正，支持原地修改。

#### 5.7 均匀性指标

校正后计算图像均匀性，评估校正效果：

$$\text{均匀性} = 1 - \frac{\sigma}{\mu} = 1 - CV$$

其中CV（Coefficient of Variation，变异系数）越小表示图像越均匀。

**计算区域**：通常取中心区域（去掉边缘10%）进行计算，避免边缘效应影响结果。

**评估标准**：

- 均匀性 > 0.95：优秀
- 均匀性 > 0.90：良好
- 均匀性 > 0.85：合格

### 6. 光谱特性标定算法

光谱特性标定用于生成相机的光谱响应标定文件，支持自动色彩校正。

#### 6.1 标定区域计算

根据镜头参数计算标定区域对应的物方视野范围：

**方法1：已知焦距和工作距离**

```

FOV_Height = 2 × Working_Distance × tan(arctan(Sensor_Height / (2 × Focal_Length)))

Region_Size = min(FOV_Width, FOV_Height) × Calib_Region_Ratio
```

### 7. Gamma校正算法

Gamma校正用于补偿人眼对亮度的非线性感知，以及校正显示器的非线性响应。

#### 7.1 Gamma编码/解码

**编码（线性RGB → sRGB）**：
$$c_{sRGB} = \begin{cases} 
12.92 \cdot c & \text{若} \, c \leq 0.0031308 \\
1.055 \cdot c^{1/2.4} - 0.055 & \text{若} \, c > 0.0031308
\end{cases}$$

**解码（sRGB → 线性RGB）**：
$$c_{linear} = \begin{cases} 
\frac{c}{12.92} & \text{若} \, c \leq 0.04045 \\
\left(\frac{c + 0.055}{1.055}\right)^{2.4} & \text{若} \, c > 0.04045
\end{cases}$$

**Gamma值**：sRGB标准使用γ≈2.2（实际为分段函数），常见的Gamma值还包括1.8（苹果系统）和2.4。

#### 7.2 应用Gamma校正

```cpp
cv::Mat GammaCorrector::apply(const cv::Mat& image, double gamma) {
    // 创建查找表
    cv::Mat lookup_table(1, 256, CV_8U);
    for (int i = 0; i < 256; ++i) {
        lookup_table.at<uchar>(0, i) = 
            static_cast<uchar>(255.0 * pow(i / 255.0, 1.0 / gamma));
    }

    // 应用查找表
    cv::LUT(image, lookup_table, result);
    return result;
}
```

**优化**：使用查找表（LUT）可以避免逐像素计算指数函数，提高处理速度。

### 8. Tone Mapping算法

Tone Mapping用于将高位深（10/12/16-bit）图像映射到8-bit显示范围。

#### 8.1 线性映射（LINEAR）

$$output = \frac{input - min}{max - min} \cdot 255$$

直接将数据范围线性映射到0-255，适用于已知数据范围的场景。

#### 8.2 百分位映射（PERCENTILE）

$$input_{clipped} = \text{clamp}(input, p_1, p_{99})$$
$$output = \frac{input_{clipped} - p_1}{p_{99} - p_1} \cdot 255$$

**优点**：自动适应动态范围，避免极端值影响（默认模式）

**实现细节**：

- p1：1%百分位，排除最暗的1%像素
- p99：99%百分位，排除最亮的1%像素
- 这种方式可以有效避免死像素或噪声导致的动态范围压缩

#### 8.3 直方图均衡化（HISTOGRAM_EQUALIZATION）

```
1. 计算图像直方图
2. 计算累积分布函数CDF
3. output = CDF[input] × 255
```

**优点**：增强对比度，使直方图均匀分布

**实现细节**：对于多通道图像，先转换为灰度图计算直方图，然后对每个通道应用相同的变换。

#### 8.4 Gamma校正映射（GAMMA）

$$output = 255 \cdot \left(\frac{input}{max}\right)^{1/\gamma}$$

**优点**：符合人眼感知特性，优化暗部细节

**常用Gamma值**：2.2（匹配sRGB标准）

### 9. 校正Pipeline

采用链式处理架构，各算法模块抽象为统一的 `ImageNode` 接口，遵循硬件 ISP 处理顺序：

**标准处理流程**：

```
硬件裸流输入（8/10/12/16-bit Raw Data）
       ↓
节点 1：DPC 坏点校正（必须在 Bayer 展开前原地修改）
       ↓
节点 2：LSC 平场校正（消除阴影与渐晕）
       ↓
节点 3：去噪模块 Denoise（消除 LSC 放大边缘信号带来的高频噪声）
       ↓
节点 4：Demosaicing 去马赛克插值（Bayer → RGB）
       ↓
节点 5：WB 白平衡增益应用
       ↓
节点 6：CCM 线性色彩空间转换
       ↓
节点 7：Gamma / Tone Mapping（高位深映射至 8-bit 显示区间）
```

| 节点          | 顺序  | 功能                              | 类名               | 处理阶段   |
| ----------- | --- | ------------------------------- | ---------------- | ------ |
| DPC         | 1   | 坏点校正（必须在Bayer展开前原地修改）           | DPCNode          | Bayer域 |
| LSC         | 2   | 平场校正（消除阴影与渐晕）                   | LSCNode          | Bayer域 |
| Denoise     | 3   | 去噪（消除LSC放大边缘信号带来的高频噪声）          | DenoiseNode      | Bayer域 |
| Demosaicing | 4   | 去马赛克插值（Bayer→RGB）               | DemosaicingNode  | 转换域    |
| WB          | 5   | 白平衡增益应用                         | WhiteBalanceNode | RGB域   |
| CCM         | 6   | 线性色彩空间转换                        | CCMNode          | RGB域   |
| Gamma       | 7   | Gamma/Tone Mapping（高位深→8-bit显示） | GammaNode        | RGB域   |

#### 节点顺序设计原则

**Bayer域处理（节点1-3）**：

- **DPC 优先**：坏点校正在原始Bayer数据上执行，确保插值前修复缺陷像素
- **LSC 居中**：平场校正消除镜头渐晕，为后续处理提供均匀基础
- **Denoise 后置**：消除LSC放大边缘信号带来的高频噪声，避免噪声传播

**RGB域处理（节点5-7）**：

- **先线性后非线性**：WB和CCM在线性空间执行，Gamma在最后进行非线性映射

#### SCM 测试数据在 Pipeline 中的应用

SCM（传感器特性表征）的输出指标是 Pipeline 各节点参数化的物理依据。以下是 SCM 测试数据具体作用于 Pipeline 各节点的完整方案：

**SCM 基础测试数据**：

| 指标          | 测得值           | 物理意义              |
| ----------- | ------------- | ----------------- |
| 暗信号         | 99.4976 DN    | OB 黑电平偏置          |
| 暗电流         | 2002.73 e-/s  | 单位时间暗信号累积         |
| 系统增益        | 1.99978 DN/e- | 电子到数字值的转换系数       |
| 暗噪声         | 29.9848 DN    | 读出噪声 RMS（约 30 DN） |
| 满阱容量        | 62258.2 DN    | 像素饱和上限            |
| 50% 信号 SNR  | 42.9617 dB    | 中间灰阶信噪比           |
| 动态范围        | 11.0209 bit   | 有效信息位深            |
| 推荐 CCM 标定曝光 | 2000 us       | CCM 标定最佳曝光时间      |
| 推荐 LSC 标定曝光 | 2000 us       | LSC 标定最佳曝光时间      |

---

**节点 0：OB 扣除（Optical Black / Black Level Subtraction）**

- **输入 SCM 数据**：暗信号 = 99.4976 DN，暗电流 = 2002.73 e-/s，系统增益 = 1.99978 DN/e-

- **具体作用**：所有后续节点依赖绝对线性空间。读取裸流后，必须全图逐像素扣除这个底噪基底。

- **短曝光（标准采集）**：直接全局减去静态底噪
  
  ```
  Raw = max(Raw - 99.5, 0)
  ```

- **长曝光或曝光时间变化**：加入暗电流漂移补偿
  
  $$
  \text{Offset} = 99.4976 + (2002.73 \times t \times 1.99978)
  $$
  
  ```
  Raw = max(Raw - Offset, 0)
  ```

---

**节点 1：DPC 坏点校正**

- **输入 SCM 数据**：暗噪声 = 29.9848 DN（约 30 DN）
- **具体作用**：设定动态统计学阈值。取代人工测试得出的 fixed threshold（如 100 或 200），利用统计学原理（如 $5\sigma$）来判定真正的坏点，而不是误伤散粒噪声。
- **判定条件**：当某像素与其邻域的差值 $\Delta > 5 \times 30 = 150\text{ DN}$ 时，才将其标记为坏点进行插值。这在 16-bit 空间中是一个非常敏锐且极难误判的安全边界。

---

**节点 2：LSC 平场校正**

- **输入 SCM 数据**：满阱容量 = 62258.2 DN，推荐 LSC 标定曝光 = 2000 us
- **标定阶段**：控制光源和 2000 us 曝光，确保捕获的亮场图像中心区域的亮度峰值严格控制在**满阱容量的 70%~80%**（约 43000~49000 DN）。绝对不能超过 62258 DN，否则发生高光截断（Clip），算出的增益图中心会呈诡异的平顶状，导致校正后中心发暗。
- **校正阶段**：应用乘法增益时，使用已扣除 OB 的线性数据，防止阴影处的黑电平被 LSC 放大产生偏色。

---

**节点 3：去噪模块 Denoise**

- **输入 SCM 数据**：系统增益 = 1.99978 DN/e-，暗噪声 = 29.9848 DN

- **具体作用**：构建精确的噪声分布模型（Noise Profile）。高端降噪算法（如 BM3D、NLM）或 AI 降噪需要知道图像在不同亮度下的噪声方差 $\sigma^2$ 才能精准抹除噪声而不伤细节。

- **噪声方差模型**：基于光子散粒噪声的泊松分布特性，对于亮度为 $I$ 的像素，其真实的噪声方差模型为：
  
  $$
  \sigma^2(I) = (\text{暗噪声})^2 + (\text{系统增益} \times I) = 30^2 + (1.99978 \times I)
  $$
  
  将这个公式喂给降噪器，它就能自适应地在暗部抑制读取噪声，在高光处抑制散粒噪声。

---

**节点 4 & 5：去马赛克（Demosaicing）& 白平衡（WB）**

- **输入 SCM 数据**：无直接输入，但极度依赖 N0 和 N2。
- **具体作用**：这两步执行纯粹的线性计算。由于 N0 完美去除了 99.5 DN 的偏置，N2 将四周的光学衰减拉平，此时计算出的 WB 增益系数（R_gain, B_gain）在画面的中心和边缘将高度一致，不会出现"中心不偏色，边缘偏品红"的工业相机常见缺陷。

---

**节点 6：CCM 线性色彩空间转换**

- **输入 SCM 数据**：50% 信号 SNR = 42.9617 dB，推荐 CCM 标定曝光 = 2000 us
- **标定阶段把控**：CCM 矩阵运算极度放大色彩噪声。报告确认 SNR > 42dB，远超 20dB 的可用门槛，说明此时算出的 3x3 矩阵不会因为暗噪干扰而发生矩阵奇异或病态。
- **防溢出保护**：将 RGB 乘以 CCM 后，可能会产生超过最大物理容量的值。这里的 Clamp 上限不是 65535，而是**满阱容量 (62258 DN) - OB (99.5 DN)**。

---

**节点 7：Gamma / Tone Mapping**

- **输入 SCM 数据**：动态范围 = 11.0209 bit，满阱容量 = 62258.2 DN
- **具体作用**：指导精确的动态范围压缩（HDR to SDR）。传感器是 16-bit 输出，但实际有效信息只有 11.02 bit（大约 2048 级有效灰阶），强行把 0~65535 线性映射到 0~255 会严重浪费对比度。
- **物理级映射范围**：映射的输入区间起点不是 0，终点不是 65535。有效物理数据区间是 **[0, 62158.7]**（已扣除 OB）。
- **百分位裁剪/直方图均衡**：只在这个物理有效的范围内进行，能最大限度地保留 42.9 dB 峰值 SNR 带来的纯净画面质感。

#### 参数定点量化（硬件化准备）

将浮点型参数乘以量化系数，四舍五入转为整型参数，供 FPGA / ISP 硬件加速器直接调用：

**量化公式**：

$$R_{gain\_int} = \text{round}(R_{gain} \times Q)$$
$$B_{gain\_int} = \text{round}(B_{gain} \times Q)$$
$$CCM_{ij\_int} = \text{round}(CCM_{ij} \times Q)$$

**量化系数选择**：

| 应用场景   | 量化系数  | 对应位数   | 适用参数             |
| ------ | ----- | ------ | ---------------- |
| 低精度场景  | 1024  | 10-bit | WB增益（$\pm 10$范围） |
| 高精度场景  | 4096  | 12-bit | CCM矩阵（$\pm 4$范围） |
| 超高精度场景 | 16384 | 14-bit | 高精度色彩校准          |

**反量化（硬件输出转浮点）**：

$$R_{gain} = \frac{R_{gain\_int}}{Q}$$
$$CCM_{ij} = \frac{CCM_{ij\_int}}{Q}$$

**量化误差分析**：

以量化系数 $Q = 4096$ 为例：

- **最小可表示值**：$1/4096 \approx 0.000244$
- **最大量化误差**：$\pm 1/2Q \approx \pm 0.012\%$
- **对DeltaE的影响**：< 0.1（可忽略）

**量化接口**：

```cpp
struct QuantizedParams {
    int Q;                      // 量化系数
    int R_gain_int;             // 量化后的R增益
    int B_gain_int;             // 量化后的B增益
    int CCM_int[3][3];          // 量化后的CCM矩阵
};

QuantizedParams Quantize(float R_gain, float B_gain, 
                         float CCM[3][3], int Q);

void Dequantize(QuantizedParams q_params,
                float& R_gain, float& B_gain,
                float CCM[3][3]);
```

#### 设计特点

- **原地修改（In-place）**：DPC节点支持直接在输入图像上修改，减少内存分配
- **双缓冲机制**：前后帧独立缓冲，支持实时采集
- **灵活组合**：可根据需求选择性启用/禁用某些节点
- **硬件对齐**：节点顺序与典型ISP流水线一致，便于算法移植
- **量化支持**：提供定点量化接口，支持FPGA/ISP硬件加速器直接调用

**Pipeline执行**：

```cpp
class ImagePipeline {
    std::vector<std::unique_ptr<ImageNode>> nodes_;

public:
    cv::Mat process(const cv::Mat& input) {
        cv::Mat result = input.clone();
        for (auto& node : nodes_) {
            result = node->process(result);
        }
        return result;
    }
};
```

### 10. 快速色彩标定算法

快速色彩标定整合了白平衡和CCM标定，提供一站式色彩校正方案。

#### 10.1 标定流程

```
1. 采集/导入色卡图像
2. 绘制24色卡ROI区域
3. 检查亮度值（G通道归一化值，建议0.55~0.65）
4. 自动调节曝光和增益（可选）
5. 选择标定模式（标准模式/参考模式）
6. 设置标定参数（白平衡方法、CCM方法、Gamma类型）
7. 执行标定，生成WB增益和CCM矩阵
8. 计算DeltaE00误差，评估标定效果
```

#### 10.2 亮度检查

```
brightness = mean(G_channel) / 255.0
```

**建议范围**：0.55 ~ 0.65（对应24色卡最后一行第二个色块的G通道数值）

**原因**：色卡色块的最佳亮度范围，既能保证足够的动态范围，又不会过曝。

#### 10.3 标准色卡参考值

使用X-Rite Classic 24色卡的sRGB D65标准参考值：

| 色块序号 | 颜色名称        | sRGB参考值         |
| ---- | ----------- | --------------- |
| 1    | Dark Skin   | (115, 82, 68)   |
| 2    | Light Skin  | (194, 150, 130) |
| 3    | Blue Sky    | (98, 122, 157)  |
| 4    | Foliage     | (87, 108, 67)   |
| 5    | Blue Flower | (133, 128, 177) |
| ...  | ...         | ...             |

#### 10.4 标定结果评估

**DeltaE00误差统计**：

- 平均DeltaE00：所有色块的平均误差
- 最大DeltaE00：误差最大的色块
- 通过率：DeltaE00 < 3.0的色块比例

**合格标准**：平均DeltaE00 < 3.0，最大DeltaE00 < 6.0
