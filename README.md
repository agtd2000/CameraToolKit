# CameraToolKit - 工业相机图像标定与校正工具

基于图像标定与校正工具，提供快速色彩标定、色彩特性标定、光谱特性标定、色彩误差计算、坏点校正、平场校正功能。

## 功能概览

| 功能模块 | 说明 |
|---------|------|
| 快速色彩标定 | 标定相机色彩参数，使相机在任意照明条件下实现标准值或参考图像的色彩效果 |
| 色彩特性标定 | 多光源环境下的白平衡和CCM标定，支持量化系数调节 |
| 光谱特性标定 | 生成光谱特性标定文件，用于相机自动色彩校正，支持出厂/用户标定模式 |
| 色彩误差计算 | 计算校正结果与参考图像的CIE DeltaE00色彩偏差 |
| 坏点校正 | 自动检测和校正传感器坏点（亮坏点/暗坏点），支持多帧统计确认 |
| 平场校正(LSC) | 镜头阴影校正，消除渐晕效应导致的中心亮边缘暗问题 |

## 新增特性

| 特性 | 说明 |
|------|------|
| 12-bit/16-bit高位深支持 | 支持处理工业相机的高位深原始图像 |
| Tone Mapping | 多种映射模式（线性/百分位/直方图均衡化/伽马）将16-bit映射到8-bit显示 |
| wxAuiManager布局 | 可停靠的三栏布局，支持浮动、拖拽调整 |
| 日志系统 | 自动记录操作、数据和错误信息，便于调试和排查问题 |
| 跨平台支持 | 支持Windows和Mac系统 |
| **校正Pipeline** | 链式图像校正处理（LSC→降噪→插值→白平衡→CCM→Gamma） |
| **内存池机制** | 预分配图像缓冲区，避免高帧率采集时的内存碎片 |
| **多线程处理** | 使用wxThread异步执行耗时算法，保持UI响应 |

## 功能详细说明

### 1. 快速色彩标定

**功能说明**：标定相机色彩参数，使相机在任意照明条件下实现标准值或参考图像的色彩效果。

**标定模式**：
- **标准模式**：使用X-Rite Classic 24色卡sRGB D65标准参考值，使相机实现标准值的色彩效果
- **参考模式**：导入自定义参考图像，使相机实现与参考图像相同的色彩效果

**操作流程**：
1. 采集/导入图像
2. 绘制24色卡ROI
3. 检查亮度值（建议范围0.55~0.65，对应24色卡最后一行第二个色块的G通道数值）
4. 若亮度不在范围内，可点击"自动"调节曝光和增益
5. 选择标定模式并导入参考（参考模式需导入参考图像）
6. 高级设置：RGB统计方法（中值法/均值法）、白平衡标定方法（平均法/解析法）、CCM标定方法（最优/最小二乘）
7. 执行标定，查看白平衡参数、CCM矩阵和DeltaE00误差

**核心算法**：
- 白平衡标定：平均法（各色块分别计算后取平均）/ 解析法（误差最小原则）
- CCM标定：3x3色彩校正矩阵，支持最优法（SVD分解）和最小二乘法
- 亮度检查：G通道归一化值，灰度换算 [255×0.55] ~ [255×0.65]

### 2. 色彩特性标定

**功能说明**：在多种光源环境下进行白平衡和CCM标定，生成色彩特性标定文件。

**标定流程**：
1. 采集各光源下24色标准色卡的色彩特性
2. 每个光源需填写名称和色温，绘制24色ROI获取当前亮度
3. 添加光源信息，更换光源重复采集
4. 设置标定参数（白平衡方法、CCM方法、标准色彩选择）
5. 预览标定结果，启用量化开关可调整白平衡和CCM的量化系数
6. 导出标定文件

**量化系数**：V5.0.0新增功能，支持对色彩校正参数进行对应倍数的量化，便于硬件运算。

### 3. 光谱特性标定

**功能说明**：生成光谱特性标定文件，用于实现相机的自动色彩校正功能。

**标定模式**：
| 模式 | 适用场景 | 复杂度 |
|------|---------|--------|
| 出厂标定 | 从零开始标定典型光源 | 标准/简易 |
| 用户标定 | 基于出厂标定增加更多光源 | 标准/简易 |

**操作复杂度**：
- **标准**：需采集各种光源在标准24色背景下的反射光谱曲线（需准备24张不同颜色卡纸）
- **简易**：仅需采集白色背景下的反射光谱曲线（仅需白色卡纸）

**限制**：光谱标定文件最多记录20种光源数据。出厂标定通常使用8种光源，用户标定可额外添加12种。

**标定区域计算工具**：
- 方法1：已知镜头焦距和工作距离
- 方法2：已知物方视野范围

### 4. 色彩误差计算

**功能说明**：计算校正结果和参考图像的色彩偏差，评估校正结果是否可用。

**支持的误差指标**：
- **DeltaE00**（CIEDE2000）：最精确的色彩差异度量，考虑了人眼视觉特性
- **DeltaE76**（CIE76）：Lab空间欧氏距离
- **DeltaE94**（CIE94）：改进的色彩差异公式

**操作流程**：
1. 导入输入图像和参考图像
2. 绘制ROI（支持24色卡ROI、140色卡ROI、自定义ROI）
3. 可选：调整ROI提取的颜色、添加自定义颜色对、传入颜色文件（*.ini格式）
4. 可选：开启sRGB色彩空间（仅当输入和参考图像均为sRGB时）
5. 点击计算，查看各颜色对的DeltaE00误差
6. 导出误差结果

**误差评估参考**：
- DeltaE00 < 1.0：人眼不可察觉
- DeltaE00 1.0~3.0：可接受范围
- DeltaE00 3.0~6.0：明显差异
- DeltaE00 > 6.0：显著差异

### 5. 坏点校正

**功能说明**：自动检测和校正相机传感器上的坏点。

**坏点类型**：
- **亮坏点**：像素亮度异常高于邻域均值
- **暗坏点**：像素亮度异常低于邻域均值

**检测参数**：
- 阈值：与邻域均值的差值标准
- 平均亮度范围：限制计入计算的图像平均灰度范围，防止过曝
- 核大小：3×3或5×5邻域

**多帧统计**：对多张图像进行坏点计算时，仅当某坏点存在于每张图像中才被确认为真实坏点。

**校正方法**：
- 邻域均值插值
- 邻域中值插值

**数据管理**：
- 支持CSV格式导入/导出坏点信息
- 支持手动编辑坏点坐标
- 支持清空坏点列表

**支持格式**：Mono 8/10/12、BayerBG8/RG8/GR8/GB8、Bayer10/12

### 6. 平场校正（LSC）

**功能说明**：镜头阴影校正（Lens Shading Correction），消除镜头渐晕效应导致的中心亮、边缘暗问题。

**标定基准方式**：
| 方式 | 说明 |
|------|------|
| 中心为基准 | 以图像中心区域亮度为基准 |
| 最亮区域为基准 | 以图像最亮区域均值为基准 |
| 目标亮度 | 以指定亮度值为基准 |

**标定参数**：
- 边缘填充系数：避免边缘出现异常色块或明暗失真
- 平滑核大小：控制增益图的平滑程度
- 通道模式：独立通道校正 / 统一增益

**暗场校正**：支持使用暗场图像减去暗电流后计算增益，提高标定精度。

**均匀性指标**：校正后计算图像均匀性（1 - 变异系数），值越接近1表示越均匀。

## 算法原理

### 1. 白平衡算法（White Balance）

白平衡算法的核心目标是调整RGB三通道增益，使白色物体在不同光照条件下呈现为中性灰色。本系统实现了多种白平衡算法，支持基于色卡的标定和自动白平衡。

#### 1.1 平均法（Average Method）

对色卡中所有色块的RGB值分别取平均，以G通道为基准计算R、B通道的增益：

```
avg_R = (R₁ + R₂ + ... + Rₙ) / n
avg_G = (G₁ + G₂ + ... + Gₙ) / n
avg_B = (B₁ + B₂ + ... + Bₙ) / n

R_gain = avg_G / avg_R
G_gain = 1.0
B_gain = avg_G / avg_B
```

**实现细节**：遍历所有色块的RGB值，分别累加后取平均值，再以G通道增益为1.0计算R、B通道的归一化增益。

**优点**：计算简单、稳健，对单个色块的异常值不敏感

**适用场景**：标准色卡标定模式，色卡颜色分布均匀

#### 1.2 解析法（Analytic Method）

基于误差最小化原则求解最优增益，最小化目标函数：

```
minimize Σ(||WB × src_i - ref_i||²)
```

当有参考颜色时，以参考颜色为目标；否则以G通道为基准：

```
R_gain = Σ(ref_R_i) / Σ(src_R_i)
G_gain = 1.0
B_gain = Σ(ref_B_i) / Σ(src_B_i)
```

**实现细节**：分别累加源颜色和参考颜色的RGB分量，按通道比值计算增益。有参考时以参考通道和为目标，无参考时以源G通道和为基准。

**优点**：利用参考信息，标定精度更高

**适用场景**：参考图像模式，需要匹配特定参考图像的色彩

#### 1.3 灰度世界算法（Gray World）

假设场景中所有颜色的平均为灰色，直接计算整幅图像的RGB均值：

```
mean = cv::mean(image)
R_gain = mean[G] / mean[R]
G_gain = 1.0
B_gain = mean[G] / mean[B]
```

**实现细节**：使用OpenCV的`cv::mean()`函数直接计算整幅图像的RGB均值，然后以G通道为基准计算增益。

**适用场景**：场景颜色分布均匀时效果较好，如室内环境

#### 1.4 完美反射算法（Perfect Reflector）

假设图像中最亮的部分为理想白色，通过直方图阈值提取高亮区域：

```
1. 将图像转换为灰度图
2. 计算灰度直方图
3. 找到亮度前ratio%的像素阈值T（默认ratio=0.1）
4. 创建二值掩膜：gray >= T
5. 计算掩膜区域内的平均RGB
6. R_gain = mean[G] / mean[R], B_gain = mean[G] / mean[B]
```

**实现细节**：从直方图的高端开始累加，直到达到总像素数的ratio%，以此确定高亮区域阈值。使用`cv::mean(image, mask)`计算掩膜区域的加权均值。

**适用场景**：场景中存在白色参考物体时效果较好

#### 1.5 RGB统计方法

在提取ROI颜色时，支持两种统计方法：

| 方法 | 实现方式 | 特点 |
|------|---------|------|
| 均值法（MEAN） | `cv::mean(region)` | 计算简单，对异常值敏感 |
| 中值法（MEDIAN） | 排序后取中间值 | 抗干扰能力更强，避免极端像素影响（默认） |

**实现细节**：中值法需要遍历ROI内所有像素，按通道分别收集像素值，排序后取中间位置的值。

#### 1.6 应用校正

```cpp
cv::Mat WhiteBalance::apply(const cv::Mat& image, const WBCorrectParams& params) {
    // 转换为浮点型
    image.convertTo(result, CV_64F);
    
    // 逐像素应用增益（注意OpenCV的BGR顺序）
    pixel[0] *= params.b_gain;  // B通道
    pixel[1] *= params.g_gain;  // G通道
    pixel[2] *= params.r_gain;  // R通道
    
    // 截断到有效范围
    cv::min(result, max_val, result);
    cv::max(result, 0.0, result);
    
    return result;
}
```

**注意**：OpenCV默认使用BGR通道顺序，因此在应用增益时需要注意通道对应关系。

### 2. 色彩校正矩阵（CCM）算法

3×3色彩校正矩阵用于将相机RGB映射到标准RGB空间，补偿相机光谱响应与理想响应的差异。

#### 2.1 数学模型

```
[R_out]   [m₀₀ m₀₁ m₀₂] [R_in]
[G_out] = [m₁₀ m₁₁ m₁₂] [G_in] + [offset]
[B_out]   [m₂₀ m₂₁ m₂₂] [B_in]
```

其中 `offset` 通常为零向量，简化为：`RGB_out = CCM × RGB_in`

**物理意义**：相机拍摄的RGB值是场景光谱与相机光谱响应函数的卷积，通过CCM矩阵可以将相机RGB线性映射到标准色彩空间（如sRGB）。

#### 2.2 Gamma预处理

在标定前对颜色进行Gamma校正，将非线性的RGB值转换为线性值，提升标定精度：

| 类型 | 公式 | 说明 |
|------|------|------|
| NONE | 无处理 | 默认模式 |
| VALUE | `RGB' = 255 × (RGB/255)^γ` | 应用指定Gamma值 |
| LRGB2SRGB | sRGB解码曲线 | 将sRGB转换为线性RGB |

**sRGB解码公式**：
```
c ≤ 0.04045 → c / 12.92
c > 0.04045 → ((c + 0.055) / 1.055)^2.4
```

#### 2.3 标定方法

给定N个颜色对 `(src_i, ref_i)`，构建线性方程组 `A × ccm_flat = b`：

- **系数矩阵A**：大小为 `3N × 9`，每行对应一个颜色分量的方程
- **目标向量b**：大小为 `3N × 1`，存储参考颜色值

```
对于第i个颜色对（已归一化到0-1）：
A[3i, 0:2]     = [src_R_i, src_G_i, src_B_i]     → R输出方程
A[3i+1, 3:5]   = [src_R_i, src_G_i, src_B_i]     → G输出方程
A[3i+2, 6:8]   = [src_R_i, src_G_i, src_B_i]     → B输出方程

b[3i]     = ref_R_i
b[3i+1]   = ref_G_i
b[3i+2]   = ref_B_i
```

**实现细节**：每个颜色对贡献3个方程（分别对应R、G、B输出通道），矩阵A的每一行只有3个非零元素，对应输入RGB值。

#### 2.4 求解方法

| 方法 | OpenCV求解器 | 特点 |
|------|-------------|------|
| 最优法（OPTIMAL） | `cv::DECOMP_SVD` | 使用SVD分解求解最小二乘，数值稳定性好，精度高（默认） |
| 最小二乘法（LEAST_SQUARES） | `cv::DECOMP_NORMAL` | 使用正规方程 `A^T A × x = A^T b` 求解，计算速度快 |

**SVD分解优势**：当系数矩阵A接近奇异或条件数较大时，SVD分解仍能稳定求解，避免正规方程的数值不稳定问题。

#### 2.5 应用校正

对每个像素执行矩阵乘法：

```cpp
// 归一化到0-1
cv::Vec3d p = pixel / 255.0;

// 应用CCM: RGB_out = CCM × RGB_in + offset
out[0] = ccm(0,0)*p[0] + ccm(0,1)*p[1] + ccm(0,2)*p[2] + offset[0];
out[1] = ccm(1,0)*p[0] + ccm(1,1)*p[1] + ccm(1,2)*p[2] + offset[1];
out[2] = ccm(2,0)*p[0] + ccm(2,1)*p[1] + ccm(2,2)*p[2] + offset[2];

// 反归一化并截断
out = clamp(out × 255.0, 0, 255);
```

#### 2.6 量化系数

支持对标定参数进行量化，便于硬件实现：

```
quantized_r_gain = round(r_gain × quant_coeff)
quantized_ccm[i][j] = round(ccm[i][j] × quant_coeff)
```

**常用量化系数**：1024（10位精度）、4096（12位精度）

### 3. CIEDE2000色彩误差算法

CIEDE2000是目前最精确的色彩差异计算公式，考虑了人眼对不同颜色区域的感知差异。算法分为三个步骤：

#### 3.1 Step 1：计算C'ab和h'ab

```
C₁'ab = √(a₁² + b₁²), C₂'ab = √(a₂² + b₂²)
C'ab_avg = (C₁'ab + C₂'ab) / 2

G = 0.5 × (1 - √(C'ab_avg⁷ / (C'ab_avg⁷ + 25⁷)))

a₁' = a₁ × (1 + G), a₂' = a₂ × (1 + G)

C₁' = √(a₁'² + b₁²), C₂' = √(a₂'² + b₂²)

h₁' = atan2(b₁, a₁'), h₂' = atan2(b₂, a₂')  (转换到0-360°)
```

**G因子**：补偿低饱和度区域的感知差异，当饱和度接近0时，G接近0.5，对a通道进行放大。

#### 3.2 Step 2：计算ΔL'、ΔC'、ΔH'

```
ΔL' = L₂ - L₁
ΔC' = C₂' - C₁'

// 处理跨越0°/360°的情况
if |h₂' - h₁'| > 180°:
    Δh' = h₂' - h₁' + 360°  (若h₂' < h₁')
    Δh' = h₂' - h₁' - 360°  (若h₂' > h₁')
else:
    Δh' = h₂' - h₁'

ΔH' = 2 × √(C₁' × C₂') × sin(Δh' / 2)
```

**色调差异处理**：色调角在0°和360°处是连续的，需要特殊处理跨越边界的情况。

#### 3.3 Step 3：计算CIEDE2000

```
L'_avg = (L₁ + L₂) / 2
C'_avg = (C₁' + C₂') / 2

// 计算色调角均值（处理跨越边界）
if |h₁' - h₂'| ≤ 180°:
    h'_avg = (h₁' + h₂') / 2
else:
    h'_avg = (h₁' + h₂' + 360°) / 2  (若h₁' + h₂' < 360°)
    h'_avg = (h₁' + h₂' - 360°) / 2  (若h₁' + h₂' ≥ 360°)

T = 1 - 0.17×cos(h'_avg - 30°) + 0.24×cos(2h'_avg) + 0.32×cos(3h'_avg + 6°) - 0.20×cos(4h'_avg - 63°)

S_L = 1 + 0.015 × (L'_avg - 50)² / √(20 + (L'_avg - 50)²)
S_C = 1 + 0.045 × C'_avg
S_H = 1 + 0.015 × C'_avg × T

R_T = -sin(2 × 30° × exp(-((h'_avg - 275°)/25°)²)) × 2 × √(C'_avg⁷ / (C'_avg⁷ + 25⁷))

ΔE₀₀ = √[(ΔL'/S_L)² + (ΔC'/S_C)² + (ΔH'/S_H)² + R_T × (ΔC'/S_C) × (ΔH'/S_H)]
```

**各参数含义**：
- **S_L**：亮度权重因子，低亮度区域权重更高，反映人眼对暗部差异更敏感
- **S_C**：饱和度权重因子，高饱和度区域权重更高
- **S_H**：色调权重因子，与色调角相关，不同色调区域的感知差异不同
- **R_T**：旋转项，补偿蓝色区域（约275°）的感知异常，当色调角接近275°时产生负向修正

#### 3.4 RGB转Lab色彩空间

```
1. sRGB解码（如果启用）：
   c ≤ 0.04045 → c / 12.92
   c > 0.04045 → ((c + 0.055) / 1.055)^2.4

2. RGB → XYZ（sRGB D65矩阵）：
   [X]   [0.4124564  0.3575761  0.1804375] [R]
   [Y] = [0.2126729  0.7151522  0.0721750] [G]
   [Z]   [0.0193339  0.1191920  0.9503041] [B]

3. XYZ归一化（D65白点）：
   X /= 0.95047, Y /= 1.00000, Z /= 1.08883

4. XYZ → Lab：
   f(t) = t^(1/3)                    (t > (6/29)^3)
   f(t) = t / (3 × (6/29)^2) + 4/29  (t ≤ (6/29)^3)
   
   L = 116 × f(Y) - 16
   a = 500 × (f(X) - f(Y))
   b = 200 × (f(Y) - f(Z))
```

**D65白点**：CIE标准照明体D65的色度坐标，代表平均日光，色温约6500K。

#### 3.5 其他误差指标

| 指标 | 公式 | 特点 |
|------|------|------|
| DeltaE76 | `√((ΔL)² + (Δa)² + (Δb)²)` | Lab空间欧氏距离，计算简单但感知不均匀 |
| DeltaE94 | `√((ΔL/S_L)² + (ΔC/S_C)² + (ΔH/S_H)²)` | 引入权重因子，比E76更精确，适用于工业色差检测 |

#### 3.6 误差评估参考

| DeltaE00范围 | 视觉感知 | 应用场景 |
|-------------|---------|---------|
| < 1.0 | 人眼不可察觉 | 专业色彩校正、高端印刷 |
| 1.0 ~ 3.0 | 轻微差异，可接受 | 一般摄影、显示器校准 |
| 3.0 ~ 6.0 | 明显差异 | 工业检测、质量控制 |
| > 6.0 | 显著差异 | 需要重新标定 |

### 4. 坏点检测与校正算法

坏点检测用于识别传感器上异常的像素，分为亮坏点（亮度异常偏高）和暗坏点（亮度异常偏低）。

#### 4.1 单帧检测

对于每个像素(x,y)，计算其与邻域均值的差值：

```
1. 计算邻域均值（排除中心像素）：
   sum = Σ(pixel(x+dx, y+dy))  for dx, dy ∈ [-k/2, k/2], (dx,dy) ≠ (0,0)
   mean = sum / (k² - 1)

2. 计算差值：diff = pixel(x,y) - mean

3. 判断坏点：
   亮坏点：diff > threshold
   暗坏点：diff < -threshold
```

**参数说明**：

| 参数 | 默认值 | 说明 |
|------|-------|------|
| kernel_size | 3 | 邻域大小（3×3或5×5） |
| threshold | 30 | 检测阈值，与邻域均值的差值标准 |
| avg_brightness_min | 0.0 | 图像平均亮度最小值，低于此值不检测 |
| avg_brightness_max | 255.0 | 图像平均亮度最大值，高于此值不检测（防止过曝） |

**实现细节**：为避免过曝图像影响检测结果，先计算整幅图像的平均亮度，若不在指定范围内则跳过检测。

#### 4.2 多帧统计检测

为了排除噪声引起的误检，对多帧图像进行统计确认：

```
1. 对每帧图像分别检测坏点，记录候选坏点集合
2. 使用集合交集操作，找出所有帧中共同存在的坏点
3. 仅当坏点在每帧中都存在时才确认为真实坏点
```

**原理**：真实坏点是传感器硬件缺陷，在所有帧中都会出现；而噪声引起的伪坏点具有随机性，只会在部分帧中出现。

**实现细节**：使用`std::set`存储每帧的坏点坐标，通过集合交集操作找到共同的坏点。

#### 4.3 校正方法

| 方法 | 公式 | 特点 |
|------|------|------|
| 邻域均值插值 | `val = Σ(neighbors) / (k² - 1)` | 计算简单，平滑效果好 |
| 邻域中值插值 | `val = median(neighbors)` | 抗干扰能力强，适合处理椒盐噪声 |

**实现细节**：

- **均值插值**：取周围8个（3×3邻域）或24个（5×5邻域）像素的平均值
- **中值插值**：收集邻域像素值，排序后取中间值，能有效避免异常像素的影响

#### 4.4 支持的像素格式

- Mono 8/10/12
- BayerBG8/RG8/GR8/GB8
- Bayer10/12

**处理方式**：对于Bayer格式图像，直接在原始Bayer数据上进行坏点检测和校正，无需先进行Demosaicing。

### 5. 平场校正（LSC）算法

平场校正（Lens Shading Correction）用于消除镜头渐晕效应，使图像亮度分布均匀。

#### 5.1 增益图计算

增益图表示每个像素需要乘以的校正系数：

```
Gain(x,y) = Reference_Brightness / Flat(x,y)
```

**标定基准方式**：

| 方式 | 计算方法 | 适用场景 |
|------|---------|---------|
| 中心为基准 | 取中心区域（半径为图像尺寸的1/8）均值 | 大部分场景，中心区域通常最亮（默认） |
| 最亮区域为基准 | 遍历图像取最大块均值 | 图像中心不一定最亮的特殊情况 |
| 目标亮度为基准 | 使用指定的目标亮度值 | 需要标准化到特定亮度值 |

**中心区域计算**：
```
center_radius = min(width, height) / 16
center_region = image[height/2 - r : height/2 + r, width/2 - r : width/2 + r]
reference_brightness = mean(center_region)
```

#### 5.2 暗场校正

当提供暗场图像时，先减去暗电流再计算增益：

```
Corrected_Flat(x,y) = Flat(x,y) - Dark(x,y)
Gain(x,y) = Reference_Brightness / Corrected_Flat(x,y)
```

**作用**：消除传感器暗电流噪声和固定模式噪声，提高标定精度

**注意**：暗场图像需要在相同曝光条件下拍摄（盖上镜头盖）

#### 5.3 增益图平滑

对增益图进行高斯平滑，避免校正后出现块状伪影：

```
Smoothed_Gain = GaussianBlur(Gain, (smooth_kernel, smooth_kernel), 0)
```

**参数说明**：

| 参数 | 默认值 | 说明 |
|------|-------|------|
| smooth_kernel | 64 | 高斯核大小，值越大平滑效果越明显 |

**实现细节**：高斯核大小必须为奇数，实际使用时会自动调整为最近的奇数。

#### 5.4 边缘填充

使用镜像反射填充边缘区域，防止边缘出现异常：

```
Padded_Gain = copyMakeBorder(Smoothed_Gain, edge_fill, BORDER_REFLECT_101)
Gain = Padded_Gain[edge_fill:-edge_fill, edge_fill:-edge_fill]
```

**参数说明**：

| 参数 | 默认值 | 说明 |
|------|-------|------|
| edge_fill | 2 | 边缘填充像素数 |

**作用**：避免边缘像素因邻域不完整导致的增益异常，使用镜像反射可以保持边缘的连续性。

#### 5.5 通道模式

| 模式 | 说明 | 适用场景 |
|------|------|---------|
| 独立通道校正 | 每个通道使用独立的增益图 | RGB图像，各通道渐晕特性不同 |
| 统一增益 | 所有通道使用相同的增益图 | 灰度图像或需要保持色彩平衡的场景 |

#### 5.6 应用校正

```
Corrected(x,y) = Image(x,y) × Gain(x,y)
```

使用逐像素乘法（`cv::Mat::mul`）实现高效校正，支持原地修改。

#### 5.7 均匀性指标

校正后计算图像均匀性，评估校正效果：

```
均匀性 = 1 - (标准差 / 均值) = 1 - CV
```

其中CV（Coefficient of Variation，变异系数）越小表示图像越均匀。

**计算区域**：通常取中心区域（去掉边缘10%）进行计算，避免边缘效应影响结果。

**评估标准**：
- 均匀性 > 0.95：优秀
- 均匀性 > 0.90：良好
- 均匀性 > 0.85：合格

### 6. 光谱特性标定算法

光谱特性标定用于生成相机的光谱响应标定文件，支持自动色彩校正。

#### 6.1 标定区域计算

根据镜头参数计算标定区域对应的物方视野范围：

**方法1：已知焦距和工作距离**

```
FOV_Width = 2 × Working_Distance × tan(arctan(Sensor_Width / (2 × Focal_Length)))
FOV_Height = 2 × Working_Distance × tan(arctan(Sensor_Height / (2 × Focal_Length)))

Region_Size = min(FOV_Width, FOV_Height) × Calib_Region_Ratio
```

**公式推导**：
- 半视场角：θ = arctan(Sensor_Width / (2 × Focal_Length))
- 物方半宽度：Working_Distance × tan(θ)
- 物方全宽度：2 × Working_Distance × tan(θ)

**方法2：已知物方视野范围**

直接输入物方视野宽度和高度，计算对应的标定区域大小。

#### 6.2 光源信息管理

#### 6.3 标定模式

### 7. Gamma校正算法

Gamma校正用于补偿人眼对亮度的非线性感知，以及校正显示器的非线性响应。

#### 7.1 Gamma编码/解码

**编码（线性RGB → sRGB）**：
```
c ≤ 0.0031308 → c × 12.92
c > 0.0031308 → 1.055 × c^(1/2.4) - 0.055
```

**解码（sRGB → 线性RGB）**：
```
c ≤ 0.04045 → c / 12.92
c > 0.04045 → ((c + 0.055) / 1.055)^2.4
```

**Gamma值**：sRGB标准使用γ≈2.2（实际为分段函数），常见的Gamma值还包括1.8（苹果系统）和2.4。

#### 7.2 应用Gamma校正

```cpp
cv::Mat GammaCorrector::apply(const cv::Mat& image, double gamma) {
    // 创建查找表
    cv::Mat lookup_table(1, 256, CV_8U);
    for (int i = 0; i < 256; ++i) {
        lookup_table.at<uchar>(0, i) = 
            static_cast<uchar>(255.0 * pow(i / 255.0, 1.0 / gamma));
    }
    
    // 应用查找表
    cv::LUT(image, lookup_table, result);
    return result;
}
```

**优化**：使用查找表（LUT）可以避免逐像素计算指数函数，提高处理速度。

### 8. Tone Mapping算法

Tone Mapping用于将高位深（10/12/16-bit）图像映射到8-bit显示范围。

#### 8.1 线性映射（LINEAR）

```
output = (input - min) / (max - min) × 255
```

直接将数据范围线性映射到0-255，适用于已知数据范围的场景。

#### 8.2 百分位映射（PERCENTILE）

```
1. 计算图像的p1和p99百分位值
2. 裁剪：input_clipped = clamp(input, p1, p99)
3. 线性映射：output = (input_clipped - p1) / (p99 - p1) × 255
```

**优点**：自动适应动态范围，避免极端值影响（默认模式）

**实现细节**：
- p1：1%百分位，排除最暗的1%像素
- p99：99%百分位，排除最亮的1%像素
- 这种方式可以有效避免死像素或噪声导致的动态范围压缩

#### 8.3 直方图均衡化（HISTOGRAM_EQUALIZATION）

```
1. 计算图像直方图
2. 计算累积分布函数CDF
3. output = CDF[input] × 255
```

**优点**：增强对比度，使直方图均匀分布

**实现细节**：对于多通道图像，先转换为灰度图计算直方图，然后对每个通道应用相同的变换。

#### 8.4 Gamma校正映射（GAMMA）

```
output = 255 × (input / max)^(1/γ)
```

**优点**：符合人眼感知特性，优化暗部细节

**常用Gamma值**：2.2（匹配sRGB标准）

### 9. 校正Pipeline

采用链式处理架构，各算法模块抽象为统一的 `ImageNode` 接口：

**处理流程**：`LSC → Denoise → Interpolation → WhiteBalance → CCM → Gamma`

| 节点 | 功能 | 类名 |
|------|------|------|
| LSC | 镜头阴影校正 | LSCNode |
| Denoise | 图像降噪 | DenoiseNode |
| Interpolation | 插值缩放 | InterpolationNode |
| WhiteBalance | 白平衡 | WhiteBalanceNode |
| CCM | 色彩校正矩阵 | CCMNode |
| Gamma | Gamma校正 | GammaNode |

**设计特点**：
- **原地修改（In-place）**：支持直接在输入图像上修改，减少内存分配
- **双缓冲机制**：前后帧独立缓冲，支持实时采集
- **灵活组合**：可根据需求选择性启用/禁用某些节点

**Pipeline执行**：
```cpp
class ImagePipeline {
    std::vector<std::unique_ptr<ImageNode>> nodes_;
    
public:
    cv::Mat process(const cv::Mat& input) {
        cv::Mat result = input.clone();
        for (auto& node : nodes_) {
            result = node->process(result);
        }
        return result;
    }
};
```

### 10. 快速色彩标定算法

快速色彩标定整合了白平衡和CCM标定，提供一站式色彩校正方案。

#### 10.1 标定流程

```
1. 采集/导入色卡图像
2. 绘制24色卡ROI区域
3. 检查亮度值（G通道归一化值，建议0.55~0.65）
4. 自动调节曝光和增益（可选）
5. 选择标定模式（标准模式/参考模式）
6. 设置标定参数（白平衡方法、CCM方法、Gamma类型）
7. 执行标定，生成WB增益和CCM矩阵
8. 计算DeltaE00误差，评估标定效果
```

#### 10.2 亮度检查

```
brightness = mean(G_channel) / 255.0
```

**建议范围**：0.55 ~ 0.65（对应24色卡最后一行第二个色块的G通道数值）

**原因**：色卡色块的最佳亮度范围，既能保证足够的动态范围，又不会过曝。

#### 10.3 标准色卡参考值

使用X-Rite Classic 24色卡的sRGB D65标准参考值：

| 色块序号 | 颜色名称 | sRGB参考值 |
|---------|---------|-----------|
| 1 | Dark Skin | (115, 82, 68) |
| 2 | Light Skin | (194, 150, 130) |
| 3 | Blue Sky | (98, 122, 157) |
| 4 | Foliage | (87, 108, 67) |
| 5 | Blue Flower | (133, 128, 177) |
| ... | ... | ... |

#### 10.4 标定结果评估

**DeltaE00误差统计**：
- 平均DeltaE00：所有色块的平均误差
- 最大DeltaE00：误差最大的色块
- 通过率：DeltaE00 < 3.0的色块比例

**合格标准**：平均DeltaE00 < 3.0，最大DeltaE00 < 6.0
